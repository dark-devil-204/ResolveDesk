# Resolve Desk

A modern support operations platform built with Next.js, Better Auth, and Drizzle ORM.

Resolve Desk is designed as a real product demo for freelance clients and startup teams who need:

- public ticket intake for customers
- internal team queue management
- role-aware organization workflows
- analytics and SLA-focused operations

## Why This Project Exists

Most demo projects only show pretty UI. Resolve Desk demonstrates full product behavior:

- customer side submission flow
- agent/admin dashboard operations
- auth and organization boundaries
- secure API patterns with validation and rate limiting

## Product Capabilities

### Public Experience

- Submit support requests from a public page
- Create real tickets by organization slug
- Receive a ticket reference immediately

### Team Workspace

- Sign in with organization-aware sessions
- Filter and search tickets by status, priority, and query
- Reassign ticket owners from real organization members
- Update ticket status and priority with validated API routes
- Add replies and internal notes in ticket detail
- Manage watchers on ticket threads

### Organization Management

- Create organization workspace
- Update organization profile and slug
- List members and invitations
- Archive organization metadata safely

### Analytics and Operations

- Queue status charting
- Priority and SLA visual indicators
- Dashboard snapshots for open and at-risk tickets

### Security and Reliability

- Better Auth session checks on protected APIs/pages
- Zod-based request validation
- Upstash Redis-backed rate limiting in proxy middleware
- Demo mode signup feature flag

## Tech Stack

- Framework: Next.js 16 (App Router, route groups)
- Runtime/UI: React 19, Tailwind CSS v4, shadcn/ui, motion
- Auth: Better Auth + organization plugin
- Database: PostgreSQL via Neon + Drizzle ORM
- State/Data: React Hook Form, Zod, TanStack Query
- Charts: Recharts
- Rate Limiting: Upstash Redis + @upstash/ratelimit

## Project Structure

High-level structure:

- app/(auth): login, signup, organization create, public submit-ticket
- app/(dashboard): overview, tickets, ticket detail, organization, members, settings
- app/api: auth, tickets, organizations, public intake APIs
- components: auth, dashboard shell, tickets, UI primitives
- services: ticket, organization, activity, message, watcher, attachment domain logic
- db: schema and drizzle client
- scripts/seed.ts: realistic demo data and demo users

## Local Setup

1. Install dependencies

```bash
bun install
```

2. Configure environment variables in .env

```env
# App
NEXT_PUBLIC_APP_URL=http://localhost:3000
BETTER_AUTH_URL=http://localhost:3000
BETTER_AUTH_SECRET=replace_with_a_long_random_secret

# Database
DATABASE_URL=postgresql://...

# Upstash rate limiting
UPSTASH_REDIS_REST_URL=https://...
UPSTASH_REDIS_REST_TOKEN=...

# Demo behavior
ENABLE_SIGNUP=false
NEXT_PUBLIC_ENABLE_SIGNUP=false
NEXT_PUBLIC_DEMO_LOGIN_ENABLED=true
NEXT_PUBLIC_DEMO_EMAIL=admin@resolvedesk.com
NEXT_PUBLIC_DEMO_PASSWORD=Admin123!
NEXT_PUBLIC_DEMO_ORG_SLUG=resolvedesk-demo
```

3. Run migrations and seed data

```bash
bun run seed
```

4. Start development server

```bash
bun run dev
```

## Key API Surfaces

Core internal routes:

- GET/POST /api/tickets
- GET /api/tickets/stats
- PATCH /api/tickets/:id/status
- PATCH /api/tickets/:id/priority
- PATCH /api/tickets/:id/assignee
- GET/POST/DELETE /api/tickets/:id/watchers
- GET /api/organizations/:organizationId
- PATCH/DELETE /api/organizations/:organizationId

Public route:

- POST /api/public/tickets

5. Reassign ticket, change status/priority, add internal note

## Security Notes

- Do not deploy with default Better Auth secret
- Keep signup disabled for public demos unless intentional
- Keep Upstash rate limiting enabled in production
- Add Turnstile on public intake for stronger anti-bot protection
